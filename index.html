The Travelling Salesman Problem is one of the most famous problems in computer science: To find the shortest path to visit each destination exactly once.
I have implemented genetic algorithm and simulated annealing in my Android Application to find (almost)-optimal solutions quickly. I was able to only use flying distance due to limitations of the free APIs that are available.
In the video demo, I was able to get a decent result for a 30 destination problem in less than 20 seconds.
Special creds. to the Project Spot

  • Date: July 2015
  • Categories: Android, Travelling Salesman, Optimization

Self-balancing Robot

I built a self balancing robot using the Arduino microcontroller platform, which runs on C/C++.
I implemented the PID (propotional-integral-derivative) control algorithm to balance the robot, as well as implemented additional libraries for sensor digital signal processing.
Other parts used include the MPU6050 motion sensor, standard 12V power supply, and motors and wheels found commonly in hobby shops.

  • Date: May 2015
  • Categories: C/C++, Arduino, Control Theory

Blackjack Simulator

I wanted to compute the longterm ROI of Blackjack given that a player plays optimally. I researched the optimal way for a player to play Blackjack and implemented a simulator in Python. Go ahead and try it in your browser.

IMPORTANT NOTE: Trinket is unable to update the console continuously. For more than 10,000 total games (~1 min runtime), please download the source code and run on your local install of Python 3.

  • Date: January 2015
  • Categories: Python, Monte Carlo method
